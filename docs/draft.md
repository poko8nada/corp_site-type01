# corp-site t1

## メモ

- 中小企業のコーポレートサイトを静的に作成し、dist以下を納品することで完結するフォーマットを構築する。
- 納品されたdist以下を国内で一般的に使われているようなレンタルサーバーや、github pagesに置くだけで、機能するホームページのイメージ。
- ページ数は、中小企業が必要になる構成をカバーできる仕組みとする。toB toCで必要そうなものは変わる想定。ただしお問い合わせページは必ず含める
- LPには対応せず、一般的な中小企業のwebサイトとする。
- 激しいアニメーションなどは使わない前提。これはwebサイトはブランディングには重要だが、ユーザーが欲する情報に対するアクセスは迅速で分かりやすくあるべきという、ユーザーファースト思想のもとである。
- 静的であるため、お知らせや記事などは頻繁に更新しない想定とし、もしそれが必要な場合は別途CMSやそれをapiで繋ぎ込む想定。
- ただし、ブラウザが直接CMS APIを叩くと…
  - APIキーがフロントエンドに露出する
  - CMS側にCORSの設定が必要
  - CMS APIが遅いと表示も遅い

```
ブラウザ → PHP（同じレンタルサーバー上）→ CMS API
↑
ここでキャッシュ・APIキー隠蔽
```

```php
<?php
// /api/articles.php

$cache_file = '/tmp/articles_cache.json';
$cache_ttl  = 300; // 5分キャッシュ

// キャッシュが有効なら返す
if (file_exists($cache_file) && time() - filemtime($cache_file) < $cache_ttl) {
    header('Content-Type: application/json');
    echo file_get_contents($cache_file);
    exit;
}

// CMSから取得（APIキーはサーバー側に隠蔽）
$res = file_get_contents('https://your-cms.io/api/articles', false, stream_context_create([
    'http' => ['header' => 'X-API-KEY: SECRET_KEY_HERE']
]));

file_put_contents($cache_file, $res);

header('Content-Type: application/json');
echo $res;
```

- 本pjでは、一旦上記は、**実装しない**想定。

- 本pjはフォーマットの作成であるため、ヘッダーフッターや、トップページのヒーローセクション（カルーセルかどうか）など、複数パターンのコンポーネントをあらかじめ用意し、それらのコンポーネントを組み合せる、入れ替えることで、パターン違いで簡単にビルドできる予定
- ただし、このrepoは最低限のフォーマットを作りながら、実際のサイトデモとして作成するため、架空の企業を想定して情報を入れる想定。
- 注入する情報（社名、住所、トップメッセージなど、サイトカラーなど）は、構造化されたjsonから読み込む方式とし、jsonの定義も作成する（複数種あってもよいし、1つにまとめてもよい、議論）
- GTM（GA4を利用するため）の模擬タグや、sitemap、その他ホームページに必要になるものは全て揃える想定
- 本PJはフォーマットとしてcloudflare pagesにデプロイすることで、ステージングもしくは最終確認を行う。
- ただし、pagesは何らかの方法で、プライベートにして、顧客と自分で最終確認を行うことができる環境を作る想定。
- 本件では、github pagesを本番環境とする。
- デザインはpencil(.penファイル)で管理する想定

## frontend design

### design concept

- シンプル
- 堅実・信頼感
- クラシック
- エレガント
- 男性的
- 高級感

### theme color

黒に近い茶をベースカラーとして背景に敷いたりするイメージ
まるで、木材のような雰囲気
場合によって、背景にそういった1枚画像を入れる、小さい改修（ドットや線）を入れて表現してもいいかも

### font

細い
明朝
セリフ

デザイン性が高く可読性が低めなフォントは、タイトルや、ヘディングでのみ。
その他は、一般的にwebサイトで用いられる上記の条件を満たしたデフォルトフォントを選定。
